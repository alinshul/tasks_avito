1. Создание задачи
Заголовок: Успешное создание задачи пользователем
Предусловия: Пользователь перешел на сайт с таск-трекером
Шаги:
1) Нажать на кнопку "Создать задачу" в верхней панели
2) В окне создания задачи заполнить обязательные поля ("Название", "Проект", "Приоритет", "Исполнитель"), при необходимости заполнить необязательное поле "Описание"
3) Нажать на кнопку "Создать" 
Ожидаемый результат: Задача создана (сообщение об успешном создании (не отображается)), можем найти ее с помощью поиска
Постусловия: Удалить тестовую задачу из списка задач (нет такой возможности в нашем случае)

2. Открытие карточки задачи (Предположила, что в этот кейс входит не только просмотр информации в карточке, но и возможность редактирования)
Заголовок: Успешное открытие карточки задачи пользователем
Предусловия: Пользователь перешел на сайт с таск-трекером
Шаги:
1) Открыть список задач
2) Нажать на задачу
3) В появившемся окне "Редактирование задачи" отредактировать задачу, если это необходимо (например, изменить статус, если она выполнена)
4) Нажать на кнопку "Обновить"
Ожидаемый результат: Задача просмотрена/отредактирована (сообщение об успешном редактировании (в нашем случаене отображается))
Постусловия: Удалить внесенные изменения в карточке задачи

3. Поиск задачи
Заголовок: Успешный поиск задачи пользователем
Предусловия: Пользователь перешел на сайт с таск-трекером
Шаги:
1) Открыть список задач
2) В поле "Поиск" ввести название задачи/имя исполнителя/название доски
3) Найти нужную задачу в появившемся списке
Ожидаемый результат: Задача найдена
Постусловия: удалить строку из поля "Поиск"

4. Переход на доску проекта
Заголовок: Успешный переход на доску пользователем
Предусловия: Пользователь перешел на сайт с таск-трекером
Шаги:
1) Открыть список задач
2) Нажать на задачу
3) Нажать на кнопку "Перейти на доску"
Ожидаемый результат: Перешли на доску проекта
Постусловия: вернуться к списку задач
